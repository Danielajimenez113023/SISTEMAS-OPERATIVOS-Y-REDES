Módulo 2

Estructura de archivos y directorios

Objetivo: Comprender cómo se organiza la información en un sistema operativo mediante archivos y carpetas.


Este módulo está diseñado para que los estudiantes aprenderán la estructura jerárquica de los archivos, el uso de rutas y la correcta organización de la información. Realizarán actividades prácticas para crear, clasificar y reconocer directorios, fortaleciendo su habilidad para gestionar de forma ordenada los recursos del equipo.

ESTRUCTURA DE ARCHIVOS Y DIRECCIONES

Una estructura de directorios es una agrupación jerárquica de carpetas que le permite almacenar y organizar sus archivos de forma lógica y eficiente.

Tipos de directorios en el sistema operatorio

  • Directorio de un solo nivel
  • Directorio de dos niveles
  • Estructura de árbol/Estructura jerárquica
  • Estructura de grafo acíclico
  • Estructura general del directorio de gráficos

Directorio de un solo nivel: todos los archivos se encuentran en el mismo directorio, lo que facilita su mantenimiento y comprensión. 

Ventajas
Organización lógica
Eficiencia
Seguridad
Copia de seguridad y recuperación
Escalabilidad
Desventajas
Existe la posibilidad de que se produzca una colisión de nombres, ya que dos archivos pueden tener el mismo nombre.
La búsqueda se volverá lenta si el directorio es grande.
Esto no puede agrupar archivos del mismo tipo.

Directorio de dos niveles:  cada usuario tiene un Directorio de Archivos de Usuario (UFD) independiente que contiene únicamente sus archivos. 

Ventajas
Puede haber más de dos archivos con el mismo nombre, lo cual sería muy útil si hay varios usuarios.
Existiría una medida de seguridad que impediría a un usuario acceder a los archivos de otro usuario.
La búsqueda de archivos resulta muy sencilla en esta estructura de directorios.
Desventajas
El usuario no puede compartir el archivo con otros usuarios.
No tienen la capacidad de crear subdirectorios.
La escalabilidad no es posible porque un usuario no puede agrupar archivos del mismo tipo.

Estructura de árbol / jerárquica: Es una estructura jerárquica de archivos similar a un árbol invertido, donde el directorio raíz se ubica en la parte superior y contiene todos los subdirectorios y archivos organizados de forma ordenada y dependiente.

Ventajas
Permite la creación de subdirectorios dentro de un directorio.
La búsqueda es más fácil.
La clasificación de archivos, diferenciando entre importantes y no importantes, se vuelve más sencilla.
Este directorio es más escalable que las otras dos estructuras de directorios explicadas.
Desventajas
Como el usuario no tiene permitido acceder al directorio de otros usuarios.
Dado que el usuario tiene la capacidad de crear subdirectorios, si aumenta el número de subdirectorios, la búsqueda puede complicarse.
Los usuarios no pueden modificar los datos del directorio raíz.
Si los archivos no caben en un directorio, es posible que haya que trasladarlos a otros.

Estructura de grafo acíclioco: permite compartir archivos o subdirectorios entre varios directorios mediante enlaces, facilitando el acceso común y la actualización compartida de los datos.

Ventajas
Se permite compartir archivos y directorios entre varios usuarios.
Buscar se vuelve demasiado fácil.
La flexibilidad aumenta gracias al acceso para compartir y editar archivos que ofrecen múltiples usuarios.
Desventajas
Debido a su compleja estructura, resulta difícil implementar esta estructura de directorios.
El usuario debe tener mucho cuidado al editar o incluso eliminar el archivo, ya que varios usuarios acceden a él.
Si necesitamos eliminar el archivo, entonces debemos eliminar todas las referencias al archivo para eliminarlo permanentemente.

Estructura general del directorio de gráficos: Un directorio puede contener rutas que regresan al punto de partida. Esto puede hacer que la navegación y la gestión de archivos sean más complejas.

Ventajas 
Más flexible que otras estructuras de directorios.
Permite ciclos, lo que significa que los directorios pueden enlazarse entre sí.
Desventajas
Su implementación resulta más costosa en comparación con otras soluciones.
Requiere recolección de basura para administrar y limpiar archivos y directorios no utilizados.
